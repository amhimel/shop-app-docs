üí≥ Payment Backend Setup

Step 1: Clone Payment Repository

git clone https://github.com/amhimel/payment_backend.git cd payment_backend

Step 2: Install Dependencies

npm install

Step 3: Configure Environment Variables

Create a .env file:

PORT=3003 MONGO_URI=mongodb+srv://your-username:[email protected]/shops STRIPE_SECRET=sk_test_your_stripe_secret_key STRIPE_WEBHOOK_SECRET=whsec_your_webhook_secret STRIPE_PUBLISHABLE_KEY=pk_test_your_publishable_key
‚ÑπÔ∏è Getting Stripe Keys:
  1. Sign up at stripe.com
  2. Go to Developers ‚Üí API Keys
  3. Copy your publishable and secret keys
  4. For webhooks, go to Developers ‚Üí Webhooks

üîî Step 4: Configure Stripe Webhooks

You need to set up a webhook endpoint in your Stripe dashboard to receive payment events.

  1. Go to Stripe Dashboard ‚Üí Developers ‚Üí Webhooks
  2. Click Add endpoint
  3. Enter your webhook URL: https://your-domain.com/webhook
  4. Select events to listen to:
    • checkout.session.completed
    • payment_intent.succeeded
  5. Copy the Webhook Signing Secret and add it to your .env file

Step 5: Start Payment Server

npm start
‚úÖ Success! Payment backend is running on http://localhost:3003

üçÉ MongoDB Configuration

üöÄ Setting Up MongoDB Atlas

  1. Go to MongoDB Atlas
  2. Create a free cluster
  3. Create a database user with read/write permissions
  4. Add your IP address to the whitelist (0.0.0.0/0 for development)
  5. Get your connection string from Connect ‚Üí Connect your application
  6. Replace <password> with your database password

üîé Search Index Setup (Product Search)

Create a search index inside MongoDB Atlas to enable product search.

  1. Open your cluster
  2. Click the Search tab
  3. Select Create Search Index
  4. Choose JSON Editor
  5. Paste the configuration below:
{
  "mappings": {
    "dynamic": true
  }
}
Name the index: shoes (or update the index name in productControllers.js)

üîß Troubleshooting

Common Issues & Solutions

1Ô∏è‚É£ Cannot connect to MongoDB

Error: MongoNetworkError or connection timeout
  • Check MongoDB Atlas IP whitelist (use 0.0.0.0/0 for development)
  • Verify connection string in .env
  • Ensure database user has correct permissions
  • Confirm MongoDB cluster is active

2Ô∏è‚É£ Token is not valid

Error: 403 Forbidden
  • Check header format: token: Bearer YOUR_TOKEN
  • Ensure JWT_SECRET matches login + verification
  • Verify token has not expired
  • Login again to generate a new token

3Ô∏è‚É£ Webhook signature verification failed

Error: Stripe webhook error 400
  • Verify STRIPE_WEBHOOK_SECRET
  • Use express.raw() middleware for webhook route
  • Confirm webhook URL in Stripe Dashboard
  • Test locally with: stripe listen --forward-to localhost:3003/webhook

4Ô∏è‚É£ File upload not working

Error: No file received
  • Ensure uploads folder exists
  • Check field name matches upload.single('profilePhoto')
  • Verify request is multipart/form-data
  • Check multer file size limits

5Ô∏è‚É£ CORS error in mobile app

Error: Access-Control-Allow-Origin error
  • Ensure app.use(cors()) is enabled
  • For specific origin: cors({ origin: 'your-app-url' })
  • Verify correct API URL in mobile config

6Ô∏è‚É£ Search not returning results

Error: Empty search or index error
  • Create MongoDB Atlas Search index
  • Ensure index name matches (shoes)
  • Wait for index to build
  • Test simple query first

7Ô∏è‚É£ Mobile app can't connect to localhost

Error: Network request failed
  • Replace localhost with computer IP
  • Use ipconfig (Windows) or ifconfig (Mac/Linux)
  • Ensure same WiFi network
  • Check firewall settings
üí° Still having issues?
  • Check server console logs
  • Run with debug mode: DEBUG=* npm start
  • Test APIs with Postman / Thunder Client
  • Review GitHub repository issues
